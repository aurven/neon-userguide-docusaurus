# Neon Glossary

| Term | Definition |
| --- | --- |
| API (Application Programming Interface) | A set of protocols and tools allowing Neon to interact programmatically with other software systems or custom integrations. |
| Binder | Side panel in Neon for accessing the candidate browser, metadata, publishing settings, version history, and annotations related to the current content. |
| Cache | Temporary data storage to speed up performance and reduce server load. |
| Cache Manager | Tool listing cached items within Neon, allowing selective cache clearing within Neon. |
| Candidate Browser | A panel that lists potential Stories, Liveblogs, or content pieces to be added or linked to a WebPage or Section. |
| Dynamic Query | A query that updates its results in real time based on filters or content changes, automatically refreshing the content displayed in a WebPage Zone as new Stories or Liveblogs match the defined criteria. |
| Embed | Adding external content (such as videos, or interactive elements) into a Story. |
| Faceted Search | A search method that allows filtering by multiple metadata categories (e.g., date, author, tags). |
| Filter Panel | Interface component for defining criteria in Queries (e.g., by section, author). |
| IDP (Identity Provider) | A system that manages user identity and authentication, often used with SSO (Single Sign-On) protocols like OIDC. |
| Live Query | Queries that search on a Site's indexes and can be created, managed as objects, published live, linked inside a WebPage, or assigned directly to a Section. They may provide dynamic or static content results depending on configuration. |
| Liveblog | A real-time, continuously updated content type similar to a Story, designed for events or breaking news. |
| Metadata | Descriptive information attached to content (such as author, publish date, tags, categories) that helps organize and classify stories. Metadata improves content discoverability on the website and search engine. |
| OIDC (OpenID Connect) | An authentication protocol used to implement Single Sign-On (SSO), allowing users to log in to Neon via an external identity provider. |
| Semantic Search | An advanced search feature that understands context and intent to provide more relevant content retrieval across Neon content. |
| Static Query | A content query based on filters that returns a fixed result set. It doesn't update automatically and only changes when manually refreshed. |
| Story | A core editorial content unit, typically a news article or report, created and managed within Neon's CMS. |
| Version | System tracking and storing all edits to content, allowing history viewing, comparison, and restoration. |
| WebPage | A digital page where you configure and organize which content, stories, liveblogs, or other elements are displayed, defining the layout and what visitors see. |
| Workflow | The defined sequence of editorial states (Draft, Review, Approved, Published) a content item passes through. |
| Zone | A specific area within a WebPage where you define which types of Stories or Liveblogs will be displayed. |
